
USE CASE: ** The system shall provide a secure login mechanism for users.
ID: UC-001

ACTORS:
- users
- user
- system
- User: a person who wants to access the secure area of the system.

PRECONDITIONS:
1. The user has a valid username and password.
2. The user has access to a web browser and internet connection.
3. The system is operational and accessible at https://the-internet.herokuapp.com/login.

MAIN FLOW:
1. The user navigates to the login page at https://the-internet.herokuapp.com/login.
2. The user enters a valid username (tomsmit) and password (SuperSecretPassword!).
3. The system verifies the credentials and checks for any invalid attempts.
4. 4. If the credentials are valid, the system displays a success message (e.g., "You logged in successfully!").
5. 5. The system grants access to the secure area.
6. **Alternative Flows:**
7. Invalid Credentials:**
8. The user enters invalid username and password.
9. The system displays an error message (e.g., "Your username or password is invalid!").
10. The user is not granted access to the secure area.
11. Forgot Password:**
12. The user clicks on the "Forgot Password" link.
13. The system redirects the user to a password reset page.
14. The user enters the required information to reset the password.

ALTERNATIVE FLOWS:
None

POSTCONDITIONS:
1. The user is logged in successfully and has access to the secure area.
2. The user can perform actions within the secure area.
3. The system has recorded the user's login activity for security purposes.
