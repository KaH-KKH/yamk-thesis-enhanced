
USE CASE: 
ID: UC-001

ACTORS:
- users
- system
- user
- User (U)
- The System (S)

PRECONDITIONS:
1. User is on the login page
2. User has a valid username and password

MAIN FLOW:
1. User enters valid username (U1)
2. User enters valid password (U2)
3. User clicks 'Login' button (U3)
4. 4. System checks username and password against database (S1)
5. 5. If credentials are valid (S2-Yes), system displays success message (S3)
6. 6. System grants access to secure area (S4)
7. 7. User can now interact with secure area functionalities
8. Alternative Flows:
9. If credentials are invalid (S2-No), system displays error message (S5)
10. User can retry login attempt

ALTERNATIVE FLOWS:
None

POSTCONDITIONS:
1. Secure area is accessible only after successful login
2. Users can logout from secure area when needed.

NOTES:
None
